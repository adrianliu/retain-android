Instructions
To add web pages for later offline reading, browse to any webpage using the normal Browser app and 'Share' the page with Retain (menu->Share). Great for subway, airplane use or other areas with low or no network connectivity. URLs in SMS text messages can be automatically downloaded to the app.

Notes
  • Nytimes Mobile site does not include the full title; appears truncated. TechCrunch mobile only put's "TechCrunch" in the title
  • Redirects (i.e. bit.ly links) will not show the effective site in the Date/Site listing, will show "bit.ly" for now.
  • Images rely on the browser's caching mechanism. Some images may not show
  • Some pages may load faster in Airplane mode due to network timeouts when in areas with low signal
  • Sites that require a login to view pages that are then Shared from the Browser app may not authenticate in Retain the first time loaded. Try re-fetching the url within Retain.
Preferences
  • Web Browser User Agent: Some web sites change appearance (or block entirely) depending on the identity of the web browser connecting to it. Use this option to change the identity to one of several types of web browsers.
  • Use default colors: Background and text color can be overridden.
  • Process URLs from SMS: Will read a URL from any incoming SMS text message. The entire body of the text must consist of only the URL and must start with 'http'. Any extra characters may cause problems. You can send URLs from an email client using your phone's email address (different for each carrier i.e. Verizon=yourPhoneNumber@vtext.com)
